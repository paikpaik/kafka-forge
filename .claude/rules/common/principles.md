# 원칙 — YAGNI, 의존성 최소화

## 필요 이상으로 확장하지 않는다

`kafka-core-project-plan.md`의 Phase 1~5(로컬 환경, 코어 모듈 v0, 신뢰성 기능, Outbox, 옵저버빌리티)는 완료됐다. 이후 기능 추가는 실제 요청받은 범위만 구현하고, 요청하지 않은 부가 기능을 미리 만들지 않는다.

## 결정하지 않은 것을 임의로 정하지 않는다

패키지 구조, 공개 API 시그니처, 의존성 추가 여부처럼 되돌리기 어려운 결정은 먼저 사용자와 확인한다. 반대로 토픽 네이밍 정규식처럼 순수 기술적이고 되돌리기 쉬운 결정은 합리적으로 제안하고 진행해도 된다.

## 의존성 최소화

`kafka-forge`는 kafkajs, zod, `@opentelemetry/api`, `prom-client` 외의 무거운 런타임 의존성(Redis 클라이언트, DB 드라이버, 특정 프레임워크)을 추가하지 않는다. 확장이 필요하면 인터페이스로 추상화하고 구현체는 소비하는 쪽에 맡긴다.
