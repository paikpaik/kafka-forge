# 원칙 — 단계별 스코프, YAGNI

## Phase 스코프를 넘어서지 않는다

`kafka-core-project-plan.md`는 Phase 1~5로 스코프를 명확히 나눠뒀다. 지금 작업 중인 Phase에 없는 기능(예: Phase 2 작업 중에 DLQ나 OTel을 미리 넣는 것)은 구현하지 않는다. 필요성이 보이면 다음 Phase 설계 논의에서 다룬다.

## 결정하지 않은 것을 임의로 정하지 않는다

패키지명, 저장소 구조, 의존성 분리 여부처럼 되돌리기 어려운 결정은 먼저 사용자와 확인한다. 반대로 토픽 네이밍 정규식처럼 순수 기술적이고 되돌리기 쉬운 결정은 합리적으로 제안하고 진행해도 된다.

## 의존성 최소화

`packages/kafka-forge`는 kafkajs, zod 외의 무거운 런타임 의존성(Redis 클라이언트, DB 드라이버, 특정 프레임워크)을 추가하지 않는다. 확장이 필요하면 인터페이스로 추상화하고 구현체는 소비하는 쪽에 맡긴다.
