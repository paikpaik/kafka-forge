# 규칙서 작성 원칙

| 원칙 | 내용 |
|------|------|
| 200줄 이하 | 파일당 200줄 이하 유지. 넘을 것 같으면 분리부터 검토 |
| 긍정형 | "하지마" 대신 "이렇게 해"로 쓴다 |
| 중복 제거 | 겹치는 내용이면 새 파일보다 기존 문서 수정을 먼저 검토 |
| 불필요 정보 제거 | Claude가 이미 아는 일반 지식(예: Kafka 개념 자체)은 넣지 않는다. 이 레포에서만 유효한 결정만 남긴다 |
| Sensor 우선 | 기계적으로 검증 가능한 규칙(lint, 타입, 테스트)은 hook/린터로 강제하고, 판단이 필요한 것만 규칙서에 남긴다 |
