# 워크플로우 — 설계 확인 후 구현, 직접 실행 유도

## 코딩 전에 설계를 먼저 확인한다

새 Phase나 구조 변경이 필요한 작업은, 코드를 짜기 전에 설계안(파일 구조, 핵심 타입/함수 시그니처, 트레이드오프)을 먼저 정리해서 사용자 확인을 받는다. 특히 저장소 구조, 패키지 경계, 의존성 방향처럼 되돌리기 어려운 결정은 반드시 먼저 확인한다.

## 새 작업은 `/harness-plans`로 기록한다

Phase 단위 작업을 시작하기 전 `.claude-plans/`에 플랜을 남기고, 완료 후에는 실행 이력을 추가한다.

## 실행은 사용자가 직접 해보게 안내한다

이 프로젝트는 학습 목적이 강하다. 코드를 작성한 뒤에는 "왜 이렇게 짰는지"를 설명하고, 사용자가 터미널에서 직접 실행해볼 수 있는 구체적인 명령어 절차를 함께 제시한다. 통합 검증처럼 여러 프로세스를 동시에 띄워야 확인 가능한 경우에만 assistant가 대신 실행해서 결과를 보여준다.
